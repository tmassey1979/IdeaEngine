# MANDATORY_OPEN_SOURCE_POLICY

## Purpose

All code generated by Dragon Idea Engine, whether for internal testing, projects, or user-submitted ideas, **must be open source**. This ensures:

``` id="os-1"
transparency
community collaboration
legal clarity
reusability
security auditability
```

No project should remain closed source unless explicitly approved under exceptional circumstances with human oversight.

---

# License Selection

Default license for all generated projects:

- **MIT License** – permissive, widely used
- Optional alternatives (configurable per project):

``` id="os-2"
Apache 2.0
GPL v3
BSD 3-Clause
```

The **License Agent** selects the appropriate license based on:

``` id="os-3"
project type
target users
compliance requirements
reuse considerations
```

---

# Agent Enforcement

All development agents must include a **license header** in every generated file.

Example MIT header:

``` text
/*
 * Copyright (c) 2026 Dragon Idea Engine
 * Licensed under the MIT License.
 */
```

The system will reject or flag any generated code that lacks a valid open source license.

---

# Repository Creation Rules

When generating a new repository:

``` id="os-4"
initialize git repo
apply selected open source license
publish to selected provider (GitHub/GitLab/etc.)
enable public access by default
```

This ensures transparency from the first commit.

---

# Component Reuse

All reusable components must also comply with open source licensing:

``` id="os-5"
component licenses are tracked in the Component Registry
dependencies with incompatible licenses are flagged
updates or modifications inherit the original license
```

---

# Automated Compliance

The Compliance Agent checks every commit and pull request:

``` id="os-6"
license headers
dependency licenses
attribution requirements
```

Violations trigger alerts and prevent code from being merged.

# Enforcement in Pi Edition

On Raspberry Pi deployments:

``` id="os-8"

automated scripts apply the license
push to public or private provider only if compliant
```

Agents running in offline mode must still include license headers and metadata to comply once connected.
