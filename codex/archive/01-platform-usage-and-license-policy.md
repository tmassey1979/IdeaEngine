## Purpose

Define how Dragon Idea Engine can be distributed, used, and monetized while maintaining open source principles and the networked agent system.

---

# Usage Models

1. **Open Source Use (Default)**
   - Users run Dragon Idea Engine locally or on Pi clusters.
   - All generated code is open source.
   - Contributions of ideas or feedback are encouraged and may be incorporated into shared projects.

2. **Commercial License / Monetization Option**
   - Users can pay for private development, proprietary outputs, or hardware-presets.
   - Optionally, Dragon Idea Engine may retain ownership of code while delivering outputs to a paying user.
   - Licensing terms clearly define rights: who owns the generated code, and under what conditions it can be shared.

---

# Contribution & Node Requirement

To encourage networked contribution:

``` id="plat-2"
All users of the platform must run an active Agent Node
OR
Submit usable ideas for processing
```

- **Agent Node contribution**: Users provide compute to run tasks, either locally (Pi) or via cloud.
- **Idea contribution**: Users submit ideas; if accepted, the generated solution is shared publicly (open source) or privately if monetized.

This establishes a **“give back” mechanism**: the more you use, the more you contribute.

---

# Idea Handling Policies

- **Publicly Accepted Ideas**:
  - Added to the shared codebase.
  - Generated projects are fully open source and available to all.

- **Privately Accepted / Monetized Ideas**:
  - Dragon Idea Engine retains ownership.
  - Paying users may receive a copy or deployment for private use.
  - Code may not be redistributed unless agreed.

- **Open Source Default**:
  - Every idea processed without private monetization is licensed under the mandatory open source policy.
